Это онлай 2д аркада по, где один игрок запускает сервер а остальные к ниму подключаются
структура проекта
termarena/
├── Cargo.toml
├── src/
│   ├── main.rs        # точка входа, запускает сервер или клиент через аргументы
│   ├── game.rs        # логика игры: движение, столкновения, объекты
│   ├── network.rs     # TCP соединения, отправка и приём сообщений
│   ├── server.rs      # управление состоянием игры, обработка клиентов
│   └── client.rs
│   └── utils.rs

План разработки 2D онлайн терминальной аркады
1. Игровая карта
    - Карта ограничена размерами (например, 40x20 или 60x30 символов).
    - Генерация лабиринта или препятствий при старте игры.
    - Определяем, где игроки могут и не могут ходить.
    - Карта хранится на сервере, клиент получает только координаты объектов и игроков.
2. Игрок
    - Структура игрока: ник, координаты, здоровье, скорость, оружие (пули, дальность, скорость).
    - Центрирование карты вокруг игрока (его персонаж всегда по центру).
    - Логика перемещения с ограничением по карте (не ходить сквозь стены).
3. Предметы
    - Разные виды предметов:
    - Восстановление/увеличение здоровья
    - Усиление оружия (скорость пули, дальность, количество)
    - Увеличение скорости игрока
    - Автоматическая генерация предметов в случайных местах.
    - Предметы исчезают через случайное время.
    - Сервер хранит все предметы и рассылает координаты клиентам.
4. Стрельба
    - Игроки могут стрелять в разные направления.
    - Пули имеют:
        1. Координаты
        2. Направление
        3. Скорость
        4. Дальность
        5. Коллизии с другими игроками и препятствиями.
        6. Сервер обрабатывает попадания и уменьшает здоровье.
5. HUD и статистика
    - Над игроком отображается ник и здоровье.
    - В углу экрана:
        - Количество игроков
        - Фраги
        - Здоровье
        - Оружие (пули/скорость/дальность)
    - 6. Сетевой слой
    - Сервер хранит состояние игры:
        - Игроки, предметы, пули
        - Стены/препятствия
    - Клиенты получают только:
        - Координаты игроков
        - Координаты предметов
        - Координаты пуль
        - Рендеринг полностью на клиенте.
7. Тайминги
    - Сервер:
        - Игровой тик (например, 20 FPS) — обновление движения, пуль, генерации предметов.
    - Клиент:
        - Отображение карты и игроков
        - Сбор input игрока и отправка команд на сервер
8. Рендеринг в терминале
    - Используем библиотеку типа crossterm для отрисовки:
    - Карта
    - Игроки
    - Пули
    - Предметы
    - HUD
    - Игрок по центру, карта смещается вокруг него.
9. Игровая логика
    - Сервер:
        - Обработка команд игроков (движение, стрельба)
        - Коллизии пуль и игроков
        - Генерация предметов
    - Клиент:
        - Получение состояния от сервера
        - Отображение карты
        - Отправка действий игрока
10. Масштабируемость
    - Сервер оптимизируется, чтобы поддерживать несколько игроков.
    - Передача данных минимальная: только координаты и состояния объектов.
